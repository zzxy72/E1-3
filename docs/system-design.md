# Mini NPU Simulator 시스템 기획서

## 1. 문서 목적

이 문서는 Mini NPU Simulator의 시스템 구조, 모듈 책임, 데이터 흐름, 확장 포인트를 정리한 기획서다.  
요구사항에 맞는 구현을 유지하면서도, 이후 기능 추가와 성능 개선이 쉬운 구조를 목표로 한다.

## 2. 설계 목표

- 요구사항의 필수 기능을 모듈 단위로 명확히 분리한다.
- 콘솔 입출력과 핵심 판정 로직을 분리해 테스트 가능성을 높인다.
- JSON 분석 모드와 사용자 입력 모드가 같은 판정 엔진을 공유하도록 설계한다.
- 보너스 과제인 패턴 생성기와 1차원 배열 최적화를 자연스럽게 추가할 수 있게 한다.
- README는 사용자용 문서로 유지하고, 구조적 설계와 기획 세부 내용은 `docs`에서 관리한다.
- 요구사항 충족 여부는 별도 체크리스트 문서에서 관리한다.

## 3. 시스템 구성

```text
main.py
└─ mini_npu/
   ├─ constants.py      : 상수와 공통 정책
   ├─ models.py         : 데이터 모델
   ├─ normalizer.py     : 라벨 정규화
   ├─ generator.py      : Cross/X 패턴 생성기
   ├─ validator.py      : 입력 및 스키마 검증
   ├─ mac.py            : 2D/1D MAC 연산
   ├─ judge.py          : 점수 비교와 판정
   ├─ performance.py    : 반복 성능 측정
   ├─ json_loader.py    : data.json 파싱
   ├─ report.py         : 결과 요약 포맷팅
   └─ console.py        : 사용자 입력 UI
```

## 4. 실행 시나리오

### 4.1 사용자 입력 모드

1. `main.py`에서 모드 선택을 받는다.
2. `console.py`가 3x3 필터 A/B와 패턴을 한 줄씩 입력받는다.
3. `validator.py`가 입력 형식과 숫자 파싱을 검증한다.
4. `judge.py`가 `mac.py`를 사용해 Cross/X 점수를 계산한다.
5. `performance.py`가 2D 방식과 1D 평탄화 방식의 평균 연산 시간을 측정한다.
6. `console.py`가 결과와 성능 분석을 출력한다.

### 4.2 패턴 자동 생성 모드

1. 사용자가 크기 `N`을 입력한다.
2. 프로그램이 해당 크기의 Cross/X 기준 필터를 자동 생성한다.
3. 사용자가 생성할 패턴 종류(Cross/X)를 선택한다.
4. 선택된 패턴이 자동 생성된다.
5. 생성된 패턴을 즉시 MAC 판정 입력으로 사용한다.
6. 같은 패턴을 성능 분석 입력으로 재사용한다.

### 4.3 JSON 분석 모드

1. `json_loader.py`가 `data.json`을 읽는다.
2. `filters`에서 `size_5`, `size_13`, `size_25` 필터를 파싱한다.
3. `patterns`의 각 케이스에서 크기와 expected 라벨을 읽고 정규화한다.
4. 각 케이스는 대응하는 크기의 필터와 비교된다.
5. `judge.py`가 판정을 수행하고 `report.py`가 PASS/FAIL을 출력한다.
6. 성능 분석은 3x3, 5x5, 13x13, 25x25 크기에 대해 수행된다.
7. 3x3은 보너스 패턴 생성기를 사용해 기준 패턴을 만들고, 5/13/25는 JSON 필터를 사용한다.

## 5. 핵심 정책

### 5.1 라벨 정규화

프로그램 내부 표준 라벨은 아래 두 개만 사용한다.

- `Cross`
- `X`

정규화 규칙:

- `+` -> `Cross`
- `cross` -> `Cross`
- `x` -> `X`

이 정책으로 JSON의 라벨 표현 차이와 내부 로직의 비교 기준을 통일한다.

### 5.2 판정 정책

- Cross 점수 > X 점수: `Cross`
- X 점수 > Cross 점수: `X`
- `abs(cross_score - x_score) < 1e-9`: `UNDECIDED`

이 정책은 부동소수점 오차 때문에 생기는 경계 상황을 안정적으로 처리하기 위한 것이다.

### 5.3 실패 처리 정책

- 사용자 입력 오류는 즉시 안내 후 재입력을 유도한다.
- JSON 분석 오류는 프로그램 전체를 종료하지 않고 케이스 단위 FAIL로 처리한다.
- 필터/패턴 크기 불일치, 필터 누락, 지원하지 않는 라벨 등은 모두 사유를 남긴다.

## 6. 보너스 기능 설계

### 6.1 패턴 생성기

`generator.py`는 임의 크기 `N`에 대해 Cross/X 패턴을 자동 생성한다.

활용 방식:

- 3x3 사용자 입력 예시 확인
- 성능 분석용 기준 패턴 생성
- 향후 자동 테스트 데이터 확장

### 6.2 1차원 배열 최적화

`mac.py`는 기본 2차원 MAC 외에 아래 기능을 추가한다.

- `flatten_matrix()`
- `compute_mac_flat()`

`performance.py`는 2D 방식과 1D(flat) 방식을 각각 반복 측정해 비교한다.  
이를 통해 메모리 접근 패턴을 단순화했을 때의 체감 차이를 같은 입력으로 확인할 수 있다.

## 7. 확장 포인트

### 7.1 입력 방식 확장

현재는 콘솔 입력, 자동 생성 입력, JSON 파일 입력을 지원하며, 구조상 아래 확장이 쉽다.

- CSV 입력 모드
- YAML 입력 모드
- 웹 API 입력 모드

이 경우 `console.py`, `json_loader.py`와 같은 입력 어댑터만 추가하면 된다.

### 7.2 필터 종류 확장

현재는 Cross와 X만 표준 라벨로 사용하지만, 별도 정규화 규칙과 필터 집합 구조를 확장하면 다음도 가능하다.

- O 패턴
- T 패턴
- 사용자 정의 필터

### 7.3 성능 실험 확장

현재는 평균 시간만 측정하지만, 다음 지표도 쉽게 추가 가능하다.

- 최소/최대 시간
- 중앙값
- 반복 횟수별 분산
- 2D/1D 속도 향상 비율
